The DOM

  • The DOCUMENT OBJECT MODEL (DOM) defines the rules by which an HTML document can be broken down:
  • Each element in an HTML document is broken down into discrete objects
  • Each object its own properties and methods
  • Due to the existence of the DOM - we can use JavaScript to manipulate and alter the HTML elements in an HTML page.

Trees & Branches

HTML has a structure where the HTML elements form a structure of parentsâ€™ nodes with children, like the branches of a tree. There is one root element: html with branches like head and body, each with their own branches. For this reason, the DOM is also called the DOM tree.


<html>
  <head>
    <title>the title</title>
  </head>
  <body>
    <div>
      <p>This is a paragraph.</p>
      <p>This is second paragraph.</p>
      <p>This is third paragraph.</p>
    </div>
  </body>
</html>

The important points about the above example tree structure is the following:

  • The <html> is the ancestor (parent) of all the other elements; and therefore: all the other elements are descendants (children) of <html>.
  • The <head> and <body> elements are not only descendants, but immediate children of <html>, as well.
  • Likewise, in addition to being the ancestor of <head> and <body>, <html> is also their parent.
  • The <p> elements are children (and descendants) of <div>, descendants of <body> and <html>, and siblings of each other <p> elements.
  • By accessing the DOM tree of a given webpage using the document object, we can dynamically and in realtime: - Access every DOM element on the page & its properties - Create new DOM elements - Remove DOM elements - Modify their properties - ...
  • These changes are NOT PERSISTENT - meaning then when you reload the webpage, the original DOM tree will be loaded.

So - there are various ways to access single HTML elements and or groups of elements:

Note:document.getElementById() returns ONLY the 1st element it encounters in the DOM tree with the matching ID. Ensure that all your elements have unique ID's (if you are using them to access in js).

Getting an Element By ID II

You can also use the more generic JavaScript method for getting hold of an element by its ID: document.querySelect(). This method takes different patterns as arguments - so to access an element by the ID requires the use of the # tag followed by the id name. Add the following to the setup() in the 2_dom.js


 console.log(document.querySelector("#one"));
Note that - the document.querySelect() function ALWAYS returns the first match found.

Getting Elements by TAG NAME

The JavaScript method for getting hold of an element(s) by the tag name is: document.getElementsByTagName(). It works in much the same way as getElementById(), except that it takes a tag name (a, ul, li, etc) instead of an ID and returns all elements in the page with the required tag.
Note: Even if there is only one element with that tag name, Javascript will return a list (array) with one element in it . Add the following to the setup() in the 2_dom.js


    console.log(document.getElementsByTagName("div"));
And then since what is being returned is an array: we use array syntax to access: 

console.log(document.getElementsByTagName("div").length);
console.log(document.getElementsByTagName("div")[0]);
If we want to NOT return an array - and for some reason only want to return the first match we could use document.querySelect() i.e.: 

console.log(document.querySelector("div"));

Getting Elements by TAG NAME II

You can also use the more generic JavaScript method for getting hold of ALL elements by TAG: document.querySelectAll(). This method like the single version: takes different patterns as arguments, and will return a list of matches. Add the following to the setup():


console.log(document.querySelectorAll("div"));
console.log(document.querySelectorAll("div").length);
console.log(document.querySelectorAll("div")[0]);

Getting Elements by CLASS NAME

The JavaScript method for getting hold of an element(s) by the tag name is: document.getElementsByClassName().
This method returns the same kind of list as getElementsByTagName(), except that you pass a class name to be matched, not a tag name.

Access HTML content (.innerHTML)

The innerHTML property holds a reference to the HTML content within a particular element, allowing you to easily access or change the content. The example below prints the HTML within the element with the id "two".Note: how the .innerHTML property outputs also all the HTML markup within the specified element.


console.log(document.getElementById("two").innerHTML);

Access TEXT content (.textContent)

To access the text content pertaining to a particular html element, use the .textContent property (as it does not parse HTML). 

console.log(document.getElementById("two").textContent);

In summary: the textContent property outputs text/plain while .innerHTML outputs text/html. * Output of type text/plain is not parsed by the browser and results in the full content displaying. * Output of the type text/html tells the browser to parse it before displaying it.

Access Attributes of elements

So - sometimes it is also necessary to access the attribute properties of a given elmement(s). The getAttribute() method is used in order to access particular value of an attribute for an element. The following will output the id and class value attributes for the element with id "five".


  console.log(document.querySelector("#five").getAttribute("id"));
  console.log(document.querySelector("#five").getAttribute("class"));

Note: If an attribute is not present for an element, the result will return null. And: if there is more than one value for a given property - the method will return all values as a STRING :


console.log(document.querySelector("#two").getAttribute("class"));
console.log(typeof(document.querySelector("#two").getAttribute("class")));

You can also access the classes associated with an element by using the classList property. This property will return an iterable list

 

console.log(document.querySelector("#two").classList);
And finally if you want to find out which attributes a given element has, you can use the method getAttributeNames(): 

console.log(document.querySelector("#five").getAttributeNames());

Access style attributes of elements

If an element(s) have inline style attributes allocated to them, you can also access those: 

console.log(document.querySelector("#one").style);
console.log(document.querySelector("#one").style.background);
Note: in this example, we access the style object and then a particular property... Here is another ex: 

console.log(document.querySelector("#six").style);
console.log(document.querySelector("#six").style.background);
console.log(document.querySelector("#six").style.width);

Modifying DOM elements

Not only can we access properties of elements - we can also alter them ... though remember the changes you make to the document are not persistent...

Parents and Children

Before doing that -> there are two properties that are super useful. The .parentElement property when applied to an HTML element will return the immediate parent of that element. The following example will return the parent of the first span element:


console.log(document.querySelectorAll("span")[0].parentElement)
Note: we can get ancestors of elements by chaining ... i.e..parentElement.parentElement returns the immediate grandparent etc ... 

console.log(document.querySelectorAll("span")[0].parentElement.parentElement)
In contrast, the .children property returns all the immediate children as an iterable list of a given element: 

console.log(document.querySelector(".wrapper_flex_box").children)
And to get the first one: 

console.log(document.querySelector(".wrapper_flex_box").children[0])

Modify the HTML content:

Ok - so now lets modify html content of a given element: 

document.querySelector("#two").children[0].innerHTML = "<h2> this is now a header</h2>";
