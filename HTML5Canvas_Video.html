What is an API?

API stands for Application Programming Interface. In essence they allow one to create applications using pre-built components and are not unique to web development, or even to scripting languages.
Websites such as Twitter (aka X), YouTube, Flickr, Twitter, Google, Amazon ... and LOTS of others provide APIs to the public so designers and developers can integrate features into their own websites (or for other purposes such as mobile or desktop applications).

One of the main goals of an API is to standardize how certain mechanics work and to simplify otherwise complicated programming tasks.
APIs are very significant in the world of HTML5 and there a number of them to explore, including the Canvas, Media Capture,Drag and Drop, Web storage, Geolocation , among others.

If you are interested in what Web API's are in development and are available to use:: here is a good starting point from Mozilla. Also useful is https://whatwebcando.today/. And note that you can always go ahead and develop your own API ;)

THE HTML 5 CANVAS

The HTML 5 canvas element creates a fixed-size drawing surface that exposes one or more rendering contexts, which are used to create and manipulate the content shown.
In this class we will focus on the 2D rendering context.
Other contexts may provide different types of rendering; for example, WebGL uses a 3D context based on OpenGL ES.

Canvas Properties & Usage:

  • You use the canvas element to create graphics "on the fly" with JavaScript
  • You must use JavaScript to actually draw the graphics.
  • The canvas element is ONLY a container for graphics
  • The canvas is initially blank.
  • To display something: a script first needs to access the rendering context and draw on it.
  • The canvas element has a method called getContext(), used to obtain the rendering context and its drawing functions
  • getContext() takes one parameter: the type of context.
  • In case of 2D the paramter value is "2D"which is equivelent to CanvasRenderingContext2D
  • The canvas element has only two attributes: width and height. These are both optional and can also be set using DOM properties & CSS.
  • When no width and height attributes are specified -> default is 300px (w) *150px (h)

To draw a rectangle, the important methods to remember are:

  • fillRect(x, y, width, height). -> Draw a filled rect
  • strokeRect(x, y, width, height) -> Draw an outlined rect
  • clearRect(x, y, width, height) -> clears/cuts out the rectangular area

*Watch out*:
The context.fill() and context.stroke() statements need to occur AFTER you draw the ellipse....

If we now try to draw a second ellipse - we would get errors in our drawing - WHY?
Because we need to specify to officially open and close the shape - : the arc() is really a specific type of path method ... which we will come to next...

Here is an example of using the beginPath() and closePath() functions: in order to define the begin and end of the shape drawing:

context.lineWidth=2; //change stroke

context.closePath();

context.fill(); // set the fill

context.stroke();//set the stroke
context.closePath();

The ONLY other primitive shape is a Path:

  • Path methods: Access more detailed descriptions here
  • beginPath() -> Creates a new path. Once created, future drawing commands are directed into the path and used to build the path up.
  • closePath() -> connects to the start
  • stroke() -> Draws the shape by stroking its outline.
  • fill() -> Draws a solid shape by filling the path's content area.
  • lineTo(x,y) -> draw a line from where we were with the last command.
  • moveTo(x, y) -> A useful function, which doesn't actually draw anything but becomes part of the path list described above. You can probably best think of this as lifting a pen or pencil from one spot on a piece of paper and placing it on the next. -> very useful if you want to draw more complex shapes.

We can go even furthur here - and implement our triangle object using a class.
This would allow us to put the animation code, setPoints(), and display() all into the class - making more much more organized code...
Lets start with a basic class with a constructorand a display() with a new js file: triangle.js:


class Triangle {
  constructor(
    xRef,
    yRef,
    lineLength,
    strokeColor,
    fillColor,
    lineWidth,
    xSpeed,
    ySpeed
  ) {
    this.xRef = xRef;
    this.yRef = yRef;
    this.lineLength = lineLength;
    this.strokeColor = strokeColor;
    this.fillColor = fillColor;
    this.lineWidth = lineWidth;
    this.xSpeed = xSpeed;
    this.ySpeed = ySpeed;

    // the points
    this.x1 = this.xRef;
    this.y1 = this.yRef;
    this.x2 = this.xRef + this.lineLength;
    this.y2 = this.yRef;
    this.x3 = this.xRef + this.lineLength / 2;
    this.y3 = this.yRef - this.lineLength;
  }

  display(context) {
    context.beginPath(); //start a path
    context.moveTo(this.x1, this.y1); //where to start drawing
    context.lineTo(this.x2, this.y2); //lineTo(where to go from last...)
    context.lineTo(this.x3, this.y3);
    context.lineTo(this.x1, this.y1);
    context.fillStyle = this.fillColor; // change the color we are using
    context.fill();
    context.strokeStyle = this.strokeColor; // change the color we are using
    context.lineWidth = this.lineWidth;
    context.stroke();
    context.closePath(); //end a path ..
  }
}

HTML 5 VIDEO

As you probably already know: the <video> HTML element embeds a media player which supports video playback in the HTML document.
You can use the <video> for audio content as well, but the <audio> element will provide a more appropriate user experience.

HTML Video and Filters

You will find many libraries and plugins etc for manipulating html 5 video content...
BUT since we are here let's give you a small background intro in case you wish to apply some simple filters etc yourself.
Also, it is useful to understand the basic concepts. We will do this in combination with the HTML 5 canvas.
Specifically: we sill use the HTML 5 <canvas> to display every frame of video (and run it in a requestAnimationFrame()) - so then we can also manipulate/change/adapt the video pixels! ...
We will still use the video element as the container to hold the source content - but then we will display the video content using the canvas context (essentially a copy).
Let's extend upon the previous example. Below the video element in the html, add a new canvas element:


 <canvas id="videoCanvas" width=640 height=360></canvas>

This will be the element that will display the modified video :) Next, let's set up the video to ALSO play in the canvas element.
In the js - add a reference to the canvas element as well as the context:


  let canvas = document.getElementById("videoCanvas");
  let context = canvas.getContext("2d");

Then - we will access each frame of the video playing and use the context.drawImage() function to render the frame to the canvas as an image :) - then we will draw a small rect on top so that you can see this is working superly;
We run this all within a requestAnimationFrame():


/*** instead of using the video object we can use the canvas **/
  requestAnimationFrame(run);
  function run() {
    context.clearRect(0, 0, canvas.width, canvas.height);
    context.save(); //like push() in p5
    context.drawImage(video, 0, 0, canvas.width, canvas.height);
    context.fillStyle = "#FFFFFF";
    context.restore();//like pop() in p5
    context.fillRect(canvas.width/2+50, canvas.height/2,50,50);
    requestAnimationFrame(run);
  }
