COURSE DESCRIPTION

The course material and outline described here is mostly derived and adapted from the course outline created by Pippin Barr who has taught the course in the past years. Programming isn’t just about knowing the syntax of your chosen programming language. It is not just about knowing that things like if statements, boolean expressions, variables, functions, arrays, operators, looping expressions etc.. exist - and need to be used correctly in order to construct a valid program. It is about having an idea - then learning how to translate this idea into a program.

This translation process requires one to break down and organize the idea into smaller parts, components - and then solve/write/implement each part in order to build a whole. The process of having an idea and turning it into a program is not easy nor straightforward. It requires time and practice and a plan. Sometimes, one can reuse code from a previous application, or use an API or library.

So, in this course we shift our focus from syntax (CART 253)to considering strategies for building software (translating ideas into programs) - and actively practise the techniques taught and discussed in class. We will also look at more advanced programming constructs: to structure/organize code in different ways, and write more efficient, modular programs. Finally, we will look at other libraries and APIs as a way to extend ones capacities and to open up new avenues for creative expression through code. Students will spend their time practicing to build smaller and larger programs, as well as using a variety of libraries and APIs suited to creative coding while producing and practicing their own original work in a studio environment.

LEARNING OBJECTIVES

After completing this course, students should be able to:

  • Confidently translate and break down a conceptual/abstract idea into a set of functional programmed parts
  • Confidently search for, find, and use JavaScript libraries and APIs in their projects
  • Leverage the power of their programming alongside libraries to create compelling interactive experiences
  • Discuss and evaluate their creative own coding projects
  • Provide constructive feedback to others on their coding projects

COURSE OUTLINE

The curriculum for CART 263 will consist of the following graded components:

  • In class studio tasks
  • Reading reflections
  • Participation
  • Two medium size projects

STRUCTURE

Class periods will be structured and organized over (4) specific activities: a) technical lectures (live coding): presenting and demonstrating various tools, frameworks and strategies for building smaller and larger programs, b) in class studio work and presentations (collaborative or individual) and c) sessions dedicated to provide students assistance and guidance with assignments and projects

The core technical infrastructure of the course will be built around tutorials and assignments which will, at a minimum, require the use of JavaScript (libraries, frameworks, API's...). Additional workshops may be scheduled throughout the semester based on student needs and the availability of resources. Students are expected to spend around 4 - 8 hours outside of class every week, in order to practice taught material as well as work on the assigned readings, assignments and projects.

REFLECTIONS

Over the course of the semester students are expected to write two (2) reflections . Students should choose two of the artists featured in FORM + CODE (1 artist per reflection). For each reflection: students need to write one paragraph (greater-than-or equal-to-three-hundred-words) on the artist (who,what,why), and one paragraph(greater-than-or equal-to-three-hundred-words) describing and discussing one of the works by said artist. The reflections need to be in PDF format - uploaded to the Student's CART 263 Github repository. Then, the student needs to submit the Github url to the reflection via the corresponding Moodle Assignment Activity. Please see the class schedule for due dates.

IN-CLASS TASKS

Over the course of the course of the semester, starting from week two, students will be assigned in-class tasks, to be completed usually in teams-of-two. Each task will need to be completed usually within the class session. It is important and required to do the work in the allocated class time, and marks may be lost otherwise. Both the instructor and TA will be available during the allocated studio sessions to assist students in the required tasks. All students (not each team) will upload and link the completed task to their CART 263 repository as well as supplying the url to the tasks via the corresponding Moodle Assignment Activity. These tasks will cover topics discussed & demonstrated in class. Any given task will have the evaluation criteria stated in its description and will be worth 4%.
